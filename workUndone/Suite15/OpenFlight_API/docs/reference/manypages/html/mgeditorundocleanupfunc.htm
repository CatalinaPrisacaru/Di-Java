mgeditorundocleanupfunc

CALLBACK NAME

mgeditorundocleanupfunc - Editor undo cleanup callback function

SYNOPSIS

void (*mgeditorundocleanupfunc) ( mgrec* db,
mgundocleanupreason cleanupReason,
void* undoData );

DESCRIPTION

This is the signature for editor undo cleanup callback functions. If your editor tool performs an undo-able action, register an undo cleanup callback function for your editor tool instance of this form.

Editor undo cleanup functions are called to notify your editor tool when it is safe to deallocate any undo data you may have allocated for your undo callback function. Undo data may be deallocated either after the undo callback function is called or after the corresponding undo entry becomes unreachable in the Edit->Undo menu. Undo entries become unreachable when their position in the undo stack becomes deeper than the maximum undo stack size set by the user preference.

This function must deallocate any undo data allocated for the corresponding undo callback.

PARAMETERS

db
the database for which the undo/redo cleanup callback is being called
cleanupReason
the reason why the undo cleanup callback is being called - refer to mgundocleanupreason for a list of reasons an undo cleanup callback function can be called
undoData
user defined undo/redo data specified when undo/redo callback was registered via mgEditorAddUndo, mgEditorAppendUndo, mgEditorAddRedo, mgEditorAddUndoForRedo, mgEditorAddUndoForCreate, mgEditorAppendUndoForCreate, mgEditorAddUndoForDelete, mgEditorAppendUndoForDelete, mgEditorAddUndoForMove or mgEditorAppendUndoForMove
this is typically the data you will deallocate
