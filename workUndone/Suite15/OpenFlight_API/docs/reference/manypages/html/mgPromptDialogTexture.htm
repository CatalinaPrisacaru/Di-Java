mgPromptDialogTexture

FUNCTION NAME

mgPromptDialogTexture - prompts the user to select one or more texture files using a standard file browser.

C SPECIFICATION

mgstatus mgPromptDialogTexture (mgguiparent,
intmode,
int*numFiles,
mgstringlist*fileList,
... );

PYTHON SPECIFICATION

mgstatus, numFiles, name1, name2, ... mgPromptDialogTexture (mgguiparent,
intmode,
< paramKey >,
< paramValue >,
... )

DESCRIPTION

mgPromptDialogTexture displays a modal dialog in which the user is prompted to select one or more texture files.

The dialog displays the directory structure using a file browser with two push buttons Open/Save and Cancel and a texture preview area. When the user selects a texture file in the file browser, the texture preview area displays a thumbnail view of the texture and lists some useful information (dimensions) about the texture selected.

mgPromptDialogTexture functions very similar to mgPromptDialogFile. The main difference is the file browser that is displayed. As described above, the file browser displayed by mgPromptDialogTexture includes a texture preview area that is not displayed by mgPromptDialogFile. For more information on mgPromptDialogTexture, see mgPromptDialogFile.

All optional parameters are passed using variable argument style and must be terminated with MG_NULL.

Unlike mgPromptDialogFile, mgPromptDialogTexture does not allow you to supply a custom dialog. For that reason, the following optional input parameters are not supported for mgPromptDialogTexture:

MPFA_RESOURCE
MPFA_DIALOGID
MPFA_DIALOGFUNC
MPFA_USERDATA
MPFA_CALLBACKMASK

If you do not specify the optional parameter MPFA_PATTERN, mgPromptDialogTexture will automatically add the standard list of texture pattern description-extension pairs supported by Creator. If you do specify MPFA_PATTERN, you can override this default list of extensions with your own. In this case, the list you specify will be the only description-extension pairs displayed in the file browser.

C NOTES

The string list returned for fileList is dynamically allocated and must be deallocated by the caller when it is no longer needed using mgFreeStringList.

PYTHON NOTES

Currently the parent parameter is reserved for future enhancement and should be specified as None. For now the dialog is created as a child of the main Creator main window. It is not necessary to free the string list returned for fileList in Python.

In addition to the custom dialog optional parameters listed above, the following optional input parameters are not supported in Python OpenFlight Script:
MPFA_PATTERNINDEX
MPFA_EXITDIRECTORY

PARAMETERS

parent
the dialog parent to attach the prompt dialog
mode
an integer value that is either MPFM_OPEN or MPFM_SAVE that specifies the mode for the file dialog
numFiles
a pointer to an integer that holds the number of files that have been selected - if the user had selected the MPFF_MULTISELECT option, then this value can be more than one else it will be at most one
fileList
a pointer to an mgstringlist object that will be allocated and filled with the names of the selected files, MG_NULL if no files selected - caller should free this list when no longer needed using mgFreeStringList
...
optional input parameters in variable argument style

RETURNS

mgstatus
Use MSTAT_ISOK to check return value for success. Success indicates that the file selected was a valid file and the user dismissed the dialog using the Open/Save push button. If function call is successful, numFiles contains the number of files that have been selected, and fileList contains the selected file name(s), otherwise both are undefined.
numFiles
The number of file names returned.
name1
The first file name returned.
name2
The second file name returned.
...
etc.

EXAMPLE

As noted above, this function is similar to mgPromptDialogFile. See mgPromptDialogFile for an example similar to that for using mgPromptDialogTexture.

Level 4
