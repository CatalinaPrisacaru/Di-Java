mgAppend

FUNCTION NAME

mgAppend - appends a node or transformation record onto the database hierarchy.

SYNOPSIS       (C and Python)

mgbool mgAppend (mgrec*parent,
mgrec*child );

DESCRIPTION

Given a parent node record parent, mgAttach attaches child as the last child of parent.

If child is a node record, the resulting hierarchy has child as the last child of parent. Note that child will retain its original children nodes. If parent and child are both fltPolygon node records, child is attached as a nested child of parent. Otherwise child is attached as a normal child of parent.

If child is a transformation record, child will be the last transformation of parent.

Alternatively, you can use mgAppend to attach a child as the last child of a parent or mgInsert to insert a node after a sibling.

PARAMETERS

parent
the parent node record
child
the node or transformation record to be linked into the hierarchy

RETURNS

Returns MG_TRUE if successful, MG_FALSE otherwise.

EXAMPLE

The following examples (in C and Python) show the differences when using mgAttach, mgAppend, and mgInsert to attach new polygons to an object in the database.

C EXAMPLE

static void MakeNewPolys (mgrec* object)
{
   mgrec* poly1 = mgNewRec (fltPolygon);
   mgrec* poly2 = mgNewRec (fltPolygon);
   mgrec* poly3 = mgNewRec (fltPolygon);
   mgAttach (object, poly1);  // poly1 becomes first child of object
   mgAppend (object, poly2);  // poly2 becomes last child of object
   mgInsert (poly1, poly3);   // poly3 is inserted after poly1
}

PYTHON EXAMPLE

def MakeNewPolys (object):
   poly1 = mgNewRec (fltPolygon)
   poly2 = mgNewRec (fltPolygon)
   poly3 = mgNewRec (fltPolygon)
   mgAttach (object, poly1)   # poly1 becomes first child of object
   mgAppend (object, poly2)   # poly2 becomes last child of object
   mgInsert (poly1, poly3)    # poly3 is inserted after poly1

ACCESS LEVEL

Level 2
