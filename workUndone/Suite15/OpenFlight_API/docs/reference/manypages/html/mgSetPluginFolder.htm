mgSetPluginFolder

FUNCTION NAME

mgSetPluginFolder - sets the plug-in runtime directory.

SYNOPSIS       (C and Python)

mgbool mgSetPluginFolder (const char*folder );

DESCRIPTION

mgSetPluginFolder sets the folder (or folders) from which plug-ins will be loaded for stand-alone application environments. Before calling mgInit you can use this function to override this location. To specify a single folder, simply pass a single path for folder. To specify more then one folder, pass a semi-colon delimited list of paths for folder.

Note that if you specify multiple plug-in folders and the Creator plug-in folder is one of them, you must specify the Creator plug-in folder as the first in the list. If you specify multiple folders and don't specify the Creator plug-in folder, the order is not important.

Since plug-ins are loaded by mgInit, you must call mgSetPluginFolder before calling mgInit. This function is ignored if called in the Creator modeling environment.

PARAMETERS

folder
the plug-in folder(s) to search

RETURNS

Returns MG_TRUE if successful, MG_FALSE otherwise. This function will fail if called after mgInit or from a plug-in in the Creator modeling environment.

C EXAMPLE

In this example, the stand-alone program uses mgSetPluginFolder to set the location of the plug-in runtime directory. In this case, the stand-alone program wants to load Creator plug-ins and its own set of plug-ins.
// set the plug-in folders
// the Creator plug-in folder must be the first in the list
// the custom plug-in folder(s) must come after
mgSetPluginFolder ("C:/Presagis/Suite14/Creator/config/plugins;D:/MyPlugins");

// has to be called before mgInit
mgInit (argc, argv)

ACCESS LEVEL

Level 1
