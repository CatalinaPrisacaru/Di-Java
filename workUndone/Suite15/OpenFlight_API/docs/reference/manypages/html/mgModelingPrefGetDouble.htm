mgModelingPrefGetDouble

FUNCTION NAME

mgModelingPrefGetDouble - gets a double precision floating point preference value from the modeling preferences.

C SPECIFICATION

mgbool mgModelingPrefGetDouble (const mgmodelingprefnameprefName,
double*prefValue,
doubledefaultValue );

PYTHON SPECIFICATION

mgbool, prefValue mgModelingPrefGetDouble (mgmodelingprefnameprefName,
doubledefaultValue )

DESCRIPTION

mgModelingPrefGetDouble retrieves a double value from a named preference key, prefName, and stores it in prefValue. If the key prefName does not exist, the value specified by defaultValue is returned.

Currently only a small subset of the modeling preference values are accessible. For a complete list, see mgmodelingprefname.

Note that the prefix of the key, either MPREFD or MPREFI, indicates whether the corresponding preference value is double or integer, respectively.

PARAMETERS

prefName
the preference key name
prefValue
storage location for the preference value
defaultValue
default value returned if prefName not found

RETURNS

mgbool
MG_TRUE if the key prefName represents a valid modeling preference and the value can be represented by a double value, MG_FALSE otherwise.
prefValue
If function is successful, prefValue contains the retrieved value, otherwise it will contain defaultValue.

EXAMPLE

The following examples (in C and Python) show how to get the Match Vertex Tolerance preference value.

C EXAMPLE

mgbool ok;
double tolerance;
ok = mgModelingPrefGetDouble (MPREFD_MATCHVERTEXTOLERANCE, &tolerance, 0.001)

PYTHON EXAMPLE

ok, tolerance = mgModelingPrefGetDouble (MPREFD_MATCHVERTEXTOLERANCE, 0.001)

ACCESS LEVEL

Level 4
