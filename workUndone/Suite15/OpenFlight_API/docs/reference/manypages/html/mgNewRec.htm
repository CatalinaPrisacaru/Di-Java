mgNewRec

FUNCTION NAME

mgNewRec - allocates a new node record.

SYNOPSIS       (C and Python)

mgrec* mgNewRec (mgcodercode );

DESCRIPTION

Given a node record code code, mgNewRec allocates and initializes a node record of the type defined by code. A new node is returned if the action is successful. The new node is an orphan, meaning that it is not attached to any other node in the hierarchy. Use mgAttach, mgAppend or mgInsert to attach the node into the database hierarchy.

Valid codes for this function include all node type record codes except fltHeader . Since fltHeader is the root of a database, use mgNewDb or mgOpenDb to create a fltHeader node type. You can also use this function to create transformation node records using the codes fltXmTranslate , fltXmScale , etc.

When a node is created by mgNewRec it is bound to the current database (see mgSetCurrentDb and mgGetCurrentDb). After you create a new node using this function you can only attach it to other nodes that are contained in the database that was current when you called mgNewRec. To eliminate the dependency on the current database, it is recommended to use mgNewRecDb instead of mgNewRec to create new nodes whenever possible.

Calling mgNewRec is equivalent to calling mgNewRecDb and passing the result of mgGetCurrentDb as the db parameter.

PARAMETERS

rcode
the code for the new record

RETURNS

Returns the pointer to the newly created node record if successful, MG_NULL otherwise.

EXAMPLE

The following examples (in C and Python) create and attach a new polygon to an object in the database.

C EXAMPLE

static void MakeNewPoly (mgrec* object)
{
   mgrec* poly = mgNewRec (fltPolygon);
   mgAttach (object, poly);  // poly is first child of object
}

PYTHON EXAMPLE

def MakeNewPoly (object):
   poly = mgNewRec (fltPolygon)
   mgAttach (object, poly)  # poly is first child of object

ACCESS LEVEL

Level 2
