mgSetFreezeLODCentersFlag

FUNCTION NAME

mgSetFreezeLODCentersFlag - sets the preference that determines if LOD (fltLod node centers are re-calculated when database files are saved by mgWriteDb, mgSaveAsDb, or mgExportDb.

SYNOPSIS       (C and Python)

void mgSetFreezeLODCentersFlag (mgboolflag );

DESCRIPTION

When an OpenFlight file is saved, the OpenFlight API can optionally re-calculate LOD centers automatically before writing the file. This can be useful if you change any geometry contained in a LOD and you do not want to re-calculate the LOD center in your application. Sometimes, however, this is not desirable. When it is not desirable there are two ways to tell the API to not re-calculate LOD centers. First, you can control this per LOD node using the fltLodFreezeCenter attribute on the fltLod node. If you set the value of this attribute on the fltLod node to MG_TRUE, the OpenFlight API will never re-calculate the center of that LOD. Second, if you call mgSetFreezeLODCentersFlag with flag set to MG_TRUE, the API will not recalculate ANY LOD center (regardless of the value of fltLodFreezeCenter on the fltLod node).

If flag is MG_FALSE (the default), LOD centers will be re-calculated for all fltLod nodes in the database whose fltLodFreezeCenter values are MG_FALSE. If flag is MG_TRUE, no LOD centers will be re-calcuated.

The default value for this preference is MG_FALSE.

PARAMETERS

flag
whether or not to freeze lod centers

ACCESS LEVEL

Level 1
