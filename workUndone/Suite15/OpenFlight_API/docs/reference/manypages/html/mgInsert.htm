mgInsert

FUNCTION NAME

mgInsert - inserts a node or transformation record into the database hierarchy.

SYNOPSIS       (C and Python)

mgbool mgInsert (mgrec*sibling,
mgrec*node );

DESCRIPTION

Given a sibling record sibling, mgInsert inserts node after sibling in the database hierarchy.

If sibling and node are both node records, the resulting hierarchy has node as the next sibling of sibling. Note that node will retain its original children nodes.

Similarly, if sibling and node are both transformation records, the resulting hierarchy has node as the next transformation after sibling.

Alternatively, you can use mgAttach to attach a child as the first child of a parent or mgInsert to insert a node after a sibling.

Note: sibling and node must either both be node records or both be transformation records.

PARAMETERS

sibling
the sibling node or transformation record
node
the node or transformation record to be linked into the hierarchy

RETURNS

Returns MG_TRUE if successful, MG_FALSE otherwise.

EXAMPLE

The following examples (in C and Python) show the differences when using mgAttach, mgAppend, and mgInsert to attach new polygons to an object in the database.

C EXAMPLE

static void MakeNewPolys (mgrec* object)
{
   mgrec* poly1 = mgNewRec (fltPolygon);
   mgrec* poly2 = mgNewRec (fltPolygon);
   mgrec* poly3 = mgNewRec (fltPolygon);
   mgAttach (object, poly1);  // poly1 becomes first child of object
   mgAppend (object, poly2);  // poly2 becomes last child of object
   mgInsert (poly1, poly3);   // poly3 is inserted after poly1
}

PYTHON EXAMPLE

def MakeNewPolys (object):
   poly1 = mgNewRec (fltPolygon)
   poly2 = mgNewRec (fltPolygon)
   poly3 = mgNewRec (fltPolygon)
   mgAttach (object, poly1)   # poly1 becomes first child of object
   mgAppend (object, poly2)   # poly2 becomes last child of object
   mgInsert (poly1, poly3)    # poly3 is inserted after poly1

Level 2
