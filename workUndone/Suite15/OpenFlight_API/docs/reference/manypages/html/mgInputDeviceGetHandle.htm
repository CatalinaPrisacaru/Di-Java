mgInputDeviceGetHandle

FUNCTION NAME

mgInputDeviceGetHandle - requests the device handle to report events.

SYNOPSIS       (C and Python)

mginputdevice mgInputDeviceGetHandle (mgplugintoolpluginTool );

DESCRIPTION

mgInputDeviceGetHandle tries to give focus to the specified plug-in input device pluginTool and returns the device handle if successful. The OpenFlight API maintains device handles to ensure that input from different devices are treated properly. A plug-in device must obtain a device handle before sending an event.

A device (say the mouse) has focus while it is in the middle of a MMSS_START - MMSS_CONTINUE - MMSS_STOP event sequence. A device must re-acquire the device handle each time it calls mgInputDeviceSendEvent. No other devices can obtain the device handle until the device that has focus reports the button state MMSS_STOP.

The device handle must not be stored, since it may become invalid some time later (after it has lost focus).

PARAMETERS

pluginTool
the input device plug-in tool

RETURNS

Returns the device handle if plug-in device can get focus, MG_NULL otherwise.

ACCESS LEVEL

Level 4
