mgSetPolyCurrentTextureMapping

FUNCTION NAME

mgSetPolyCurrentTextureMapping - sets the texture mapping attribute in a fltPolygon or fltMesh record to the current modeling texture mapping.

SYNOPSIS       (C and Python)

mgbool mgSetPolyCurrentTextureMapping (mgrec*db,
mgrec*rec,
intlayer );

DESCRIPTION

Given a fltPolygon or fltMesh record, rec, this function sets its texture mapping attribute for the given layer according to the current modeling texture mapping selected by the user in Creator for database db.

The parameter layer is a number between 0-7, representing the layer you want to set and corresponds to the attributes:
0 : fltPolyTexmap
1 : fltLayerTexmap1
2 : fltLayerTexmap2
3 : fltLayerTexmap3
4 : fltLayerTexmap4
5 : fltLayerTexmap5
6 : fltLayerTexmap6
7 : fltLayerTexmap7

If you specify any other value for layer, the function will fail.

PARAMETERS

db
the database whose current modeling texture mapping will be used
rec
the fltPolygon or fltMesh record on which to set the texture mapping attribute
layer
the layer number 0..7

RETURNS

Returns MG_TRUE if successful, MG_FALSE otherwise.

C EXAMPLE

mgrec* poly;
mgbool status;

poly = mgNewRec (fltPolygon);
// set texture mapping attribute for layer 0 to the
// current texture mapping in the palette
status = mgSetPolyCurrentTexture (db, poly, 0);

ACCESS LEVEL

Level 4
