mgTextGetDMS

FUNCTION NAME

mgTextGetDMS - retrieves double precision floating point value represented in DMS (degrees, minutes, seconds) format from text a control.

C SPECIFICATION       (Not Available For Python)

mgstatus mgTextGetDMS (mgguicontrol,
double*dmsVal,
const char*fmtString );

DESCRIPTION

mgTextGetDMS attempts to interpret the text value displayed in the specified text control as a double precision floating point value. The text is assumed to be formatted as a DMS string as described below.

Depending on the fmtString you specify, you can specify how you expect the text in the control to be formatted. You must specify either MDFMT_LAT or MDFMT_LON as described here.

If you specify MDFMT_LAT in fmtString, the text in the control is expected to be a latitude measurement expressed in the form: DD~MM'SS"H, where DD is the number of degrees, MM is the number of minutes, SS is the number of seconds and H is either 'N' or 'S' depending on whether the value lies in the northern or southern hemisphere, respectively.

If you specify MDFMT_LON in fmtString, the text in the control is expected to be a longitude measurement expressed in the form: DD~MM'SS"H, where DD is the number of degrees, MM is the number of minutes, SS is the number of seconds and H is either 'E' or 'W' depending on whether the value lies in the eastern or western hemisphere, respectively.

By convention, values entered in the northern and eastern hemispheres are returned as positive (0.0 .. 180.0). Values entered in the southern and western hemispheres are returned as negative (-180 .. 0.0).

PARAMETERS

control
the text control
dmsVal
address of double value to receive numeric value in decimal degrees from text control
fmtString
latitude/longitude formatting string

RETURNS

Use MSTAT_ISOK to check return value for success. If function is successful, dmsVal contains the corresponding numeric value in decimal degrees, otherwise dmsVal is undefined.

ACCESS LEVEL

Level 4
